\section*{Exercise 2 - Cache mechanics}
cache size: 32KB \\
block size: 64 bytes \\
number of sets: 32KB / 16bytes = 1024 sets \\
because of sizeof(int) = 4, 4 integers fit into one block. \\
address of src matrix: 0x00 \\
address of dest matrix assuming size 64x64: 0x4000 ($64*64*4=16384$) \\
address of dest matrix assuming size 96x64 = 0x6000 ($96*64*4=25576$) \\
Number of bits for block: 4 \\
Number of bits for set: 10\\
\subsection*{(a)}
\subsubsection*{(i)}
with 4 integers per block and 64 ints per row there are 16 blocks to read/write per row. 0x00 (src) and 0x4000 (dest) map to the same cache set. we therefore know, that they'll use the same blocks for the dest[i] and src[i] row.  \\ Assuming the used blocks are numbered from 0 to 15 the access pattern will be the following: \\

\begin{tabular}{|c|c|c|c|}

We see that for the first 8 lines we have a MHHH miss/hit pattern for both src and dest. Then in the last 8 lines we just have hits because all blocks have ben read to the cache. \\
For each row we therefore have 16 misses and $64+48=112$ hits. \\ 
This gives a cache miss rate of $\underline{16/128=0.125}$
